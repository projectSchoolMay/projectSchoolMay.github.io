Что такое вёрстка сайтов, и зачем она нужна?

Вёрстка веб-страниц (сайтов) — это создание некой структуры гипертекстового документа на основе HTML-разметки, c использованием таблиц стилей или же другим языком стилизацией самого HTML-документа.

Этапы вёрстки:

  • Создаём новый проект
  • Строим HTML-структуру
  • И стилизуем этот HTML-документ
  • Передаём свой проект BACKEND разработчику.

Какой должна быть вёрстка сайтов:

  • Конечно же вёрстка сайтов должна быть адаптивной или другим языком должна быть подстроена под все устройства, то есть на компьютере помещается больше контента, чем на телефоне, соответственно на телефоне как минимум нужно уменьшать шрифт текста,чтобы он уместился в экран.
  • И обязательным фактором является читабельность кода, и оптимизированность, чтобы наш сайт работал быстрее. Разберём пример кода на JavaScript
  • И не мало важный фактор - это семантическая вёрстка, для этого обратимся к тегам HTML То есть есть обычный тег div, он может использоваться много раз в веб-вёрстке, и есть семантические теги, и допустим наш пользователь сайта незрячий, и для того чтобы ориентироваться по сайту,придумали семантические теги допустим при наведении на тег header или при клике на него, у незрячего пользователя будет звук по типу: "Шапка", "Header".
  • Также в наше время продвигаются технологии и в вёрстке сайтов появляются новые возможности, которые во многих браузерах не видны,т.к. кроссбраузерность у них пока маленькая или другим языком - поддержка всех браузеров.
    Поэтому во многих ситуациях приходится использовать какие-то старые возможности чтобы кроссбраузерность была больше.
  • Также вёрстка должна быть "Гибкой", то есть при изменении контента, она должна не "ломаться", то есть чтобы контент отображался хорошо при маленьком количестве текста и при большом.

Что такое FRONTEND-разработчик и что он делает?

FRONTEND-разработчик занимается разработкой графического интерфейса — той части приложений, которую видит пользователь. Они превращают макет, созданный веб-дизайнером, в функциональный и удобный пользовательский интерфейс. Корректное отображение полей и блоков, работающие кнопки и формы для ввода данных — всё, с чем сталкивается пользователь в браузере, находится в зоне ответственности фронтендеров.

На чём пишется FRONTEND?

FRONTEND пишется на HTML CSS JS
  • HTML - это язык гипертекстовой разметки, то есть весь текст, контент и все HTML-теги, это всё язык гипертекстовой разметки.
  • CSS - это таблица стилей, с помощью которой стилизуется сам HTML-документ, то есть цвет текста, размер шрифта и так далее..
  • JavaScript - это уже полноценный язык программирования - который отвечает за разные выпадающие меню разные подменю, плавную прокрутку страницы и слайдеры.

Что такое BACKEND-разработчик и что он делает?

BACKEND-разработчик отвечает за внутреннюю функциональность сайта. Бэкенд-разработчик работает с сервером — невидимой частью сайта, где хранится вся информация. Он создает базы данных и управляет ими, проводит интеграции с внешними сервисами и занимается всем, что находится «под капотом» сайта. Работу бэкенд-разработчика обычный пользователь увидеть не может, но именно благодаря ей сайт функционирует.

На чём пишется BACKEND?

В BACKEND разработке не как во FRONTEND, потому что есть выбор на каком языке писать: Cамые известные Python (Django),PHP,JAVA
  • Python - это ООП язык программирования, который широко используется в интернет-приложениях, разработке программного обеспечения, Data Science и машинном обучении.Для создания BACKEND используется фреймворк django или Flask. Они отличаются мощностью, возможностями и синтаксисом.На django был написан youtube.
  • PHP - это C-подобный скриптовый язык общего назначения, интенсивно применяемый для разработки веб-приложений. Самый слабый по производительности язык из приведенного списка, но на нем написаны 80% сайтовю
  • Строго типизированный ООП язык программирования общего назначения, разработанный компанией Sun Microsystems. Это самый мощный язык програмирования для создания сайтовю

Дизайн сайтов, что это?

Дизайн сайтов или же проектирование макета сайта - это проектирование пользовательских интерфейсов только в виде макета, который передаётся верстальщику и верстальщик в дальнейшем с ним работает. Другими словами, есть человек "дизайнер" и он рисует веб-сайт в программе и дальше верстальщик делает этот "рисунок" веб-сайтом.
Макет - это модель объекта в уменьшеном виде, грубо говоря чертёж.

Сравнение FRONTEND и BACKEND

FRONTEND

  • Если человек любит строить некую структуру, стилизовать эту структуру, то FRONTEND ему понравится
  • Во FRONTEND'e довольно легко начать писать код, достаточно скопировать прошлый проект и очистить весь код.
  • Процесс обучения не настолько сложный, главное понимать, что ты делаешь и зачем. В то время в BACKEND нужно определиться, в какую сферу ты будешь изучать.
  • Как по-моему мнению, писать FRONTEND код очень интересно, ты строишь структуру изменяешь её, стилизуешь, в общем чувствуешь себя дизайнером.

BACKEND

  • Если человек любит решать логические задачи на протяжении часа и больше, то BACKEND скореев всего ему понравится
  • В BACKEND вы будете писать код на протяжении часа и результат увидете только под конец.
  • Если появится ошибка, то придется проверять весь код, для того чтобы исправить ошибку
  • И неплохой плюс в том, что зарплата выше чем у FRONTEND-разработчиков.
